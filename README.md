# Task 5 Web

Решение выполнено на PHP без сторонних библиотек. Используются встроенные возможности `PDO`, `session`, `password_hash()` и Cookies.

## Что реализовано

- серверная валидация всех полей формы;
- схема `POST -> redirect -> GET` для ошибок и повторного заполнения формы;
- сохранение введённых данных в Cookies для неавторизованного пользователя;
- первичная отправка анкеты в MySQL / MariaDB;
- автоматическая генерация уникального логина и пароля после первой успешной отправки;
- сохранение в БД только хеша пароля;
- вход по логину и паролю через PHP-сессию;
- загрузка ранее сохранённых данных после авторизации;
- редактирование и перезапись существующей анкеты авторизованным пользователем;
- отдельная таблица учётных данных `submission_accounts`.

## Файлы

- `index.php` — форма, авторизация, сессия, валидация, сохранение и обновление данных;
- `styles.css` — оформление страницы и блока авторизации;
- `schema.sql` — создание БД и таблиц.

## Подготовка БД

1. Выполните `schema.sql`.
2. Если база из предыдущего задания уже существует, важно создать новую таблицу `submission_accounts`.
3. При необходимости измените настройки подключения в начале `index.php`:
   - `DB_HOST`
   - `DB_PORT`
   - `DB_NAME`
   - `DB_USER`
   - `DB_PASSWORD`

## Запуск

Если PHP установлен локально:

```bash
php -S localhost:8000
```

Откройте [http://localhost:8000](http://localhost:8000).

## Поведение

1. Неавторизованный пользователь заполняет и отправляет форму.
2. После успешного сохранения система один раз показывает выданные логин и пароль.
3. По этим данным пользователь входит в систему.
4. После входа страница загружает анкету из БД и позволяет обновить её содержимое.
